## refactor: improve docker-compose organization, consistency, and security

This PR improves the docker-compose.yml configuration for better maintainability, consistency, and security. Changes include:

### Organization & Readability 
- Added section comments to group related services
- Improved logical ordering

### Missing Configurations 
- Added explicit container_name to all services
- Added missing network configurations
- Added proper depends_on declarations

### Consistency Improvements 
- Standardized all path variables to use USERDIR consistently
- Fixed volume syntax formatting
- Standardized environment variable quoting

### Security Enhancements 
- Tightened TRANSMISSION_RPC_HOST_WHITELIST for better network isolation.

### Maintenance Improvements 
- Updated watchtower to maintained fork
- Added default values for optional ports
- Added proper service startup order
